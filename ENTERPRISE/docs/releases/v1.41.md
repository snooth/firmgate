# Release v1.41

Paste into your Git host’s **v1.41** release notes if you like. Compared to **v1.40**, this tag is a single commit (`release v1.41`) that carries the following:

- **RBAC bootstrap** — Centralised helpers (`ensure_permission_catalog`, `ensure_builtin_roles`, `apply_standard_power_permission_defaults`) replace large inline blocks in app startup and `seed_data.py` so permission rows and Standard / Power defaults stay consistent.
- **Access control API** — `GET /api/roles` ensures the catalog and built-in roles, then returns **roles and permissions together** so the Administration → Roles matrix works on older DBs. `GET /api/permissions` ensures the catalog exists before listing.
- **`admin.all` on Standard / Power** — API rejects assigning **`admin.all`** to Standard / Power (and legacy viewer / editor role names). The Access Control UI disables **`admin.all`** on those cards and surfaces **missing permission** rows when the DB is out of sync.
- **Documents: `admin_only` and shares** — Admin-only folders stay restricted, but **owners**, **portal admins**, and users with an **explicit share** on the folder or an ancestor can still access them. Internal **share grants** can satisfy list / read / versions / write when RBAC would otherwise block non-owners; write requires a write-level grant.
- **Shares list view** — At the Shares root, shows **“Shared with you”** and **“Shared by you”**; breadcrumb **Shares**; sorting respects filtered payloads for favourites / personal / shares.
- **Admin UI copy & cache-bust** — Access Control description updated; `admin.js` cache query **v6 → v7** on admin templates.
- **Housekeeping** — `version` file set to **v1.41**; adds **`Intranet.code-workspace`**.
